## Currently Building — Strumm

</div>

> **A music player that understands your taste.**
> Replay your listening history, visualize your sound DNA, and discover tracks that actually fit your vibe.

**Strumm** is a full-featured music platform — built as a Progressive Web App with offline support, AI-driven recommendations, and a listening experience that respects your attention.

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   Live Demo  →  strumm.me                                │
│   Source     →  github.com/Agnibha007/strumm             │
│   Stack      →  React · Node.js · MongoDB · PWA · CSS3   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**What makes it different** — rather than dumping a library of tracks, Strumm analyzes your patterns, profiles your sound DNA using collaborative filtering on listening history, and surfaces recommendations that feel intentional. Not algorithmic noise. Designed as a microservices architecture with a React frontend, Node.js API gateway, and MongoDB for user data and playlist persistence.

## Engineering Philosophy

</div>

<br>

**Write code for humans first.** Machines interpret anything — but the next engineer reading your code at 2 AM needs clarity. I optimize for maintainability over cleverness.

**Ship before it's perfect.** Perfect is a moving target. I build a solid foundation, launch, then iterate based on real usage. Users don't care about your roadmap — they see what you ship.

**Every pixel has a purpose.** Whether it's the spacing between two buttons or the timing of a hover transition, nothing in the UI should feel accidental. Detail is not detail. It *is* the design.

**Choose simple over easy.** Simple requires discipline. It means fewer dependencies, cleaner abstractions, and systems that don't break when you look at them wrong. Harder to build. Easier to maintain.

**The best code is deleted code.** Every feature, every abstraction, every dependency comes with a cost. I ship less code than most people — and what remains works harder.
